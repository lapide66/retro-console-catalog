Recomendacoes para imagens do projeto

Objetivo
- Padronizar as imagens para web e mobile.

Padrao visual
- Proporcao: 4:3
- Tamanho base recomendado: 1200x900 px
- Console centralizado, com margem visual ao redor
- Evitar cortes agressivos
- Manter o mesmo estilo de enquadramento para todos os consoles

Fundo
- Preferir um unico padrao para todas as imagens
- Pode ser transparente ou neutro
- Evitar misturar fundos muito diferentes entre os consoles

Formato
- Preferencia: WebP
- Alternativa segura: PNG
- Usar JPG apenas se a imagem tiver fundo solido e precisar de arquivo menor

Peso do arquivo
- Ideal: ate 250 KB por imagem
- Aceitavel: ate 400 KB por imagem

Nomenclatura
- Usar kebab-case
- Exemplo: atari-2600.webp
- Exemplo: playstation-2.webp

Fluxo sugerido
- Separar a imagem original
- Ajustar para 1200x900 px
- Padronizar enquadramento e fundo
- Exportar no formato final
- Salvar com nome padronizado

Observacao
- Como o projeto usa cards e detalhe em 4:3, preparar todas as imagens nesse padrao evita retrabalho.
